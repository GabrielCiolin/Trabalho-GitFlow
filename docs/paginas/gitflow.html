GitFlow

1. Git flow init

Inicializa o git flow no seu projeto



2. Git flow feature start

Inicia uma nova funcionalidade, irá criar um branch (baseado no develop) e automaticamente irá dar uma checkout nesse branch

git flow feature start MINHA_FUNCIONALIDADE

3. git flow feature finish

Finaliza uma funcionalidade, esse comando irá executar as seguintes ações: Merge com a branch develop; Deleta a branch funcionalidade; Checkout na branch develop; Para finalizar uma funcionalidade, você deve executar esse comando dentro do branch funcionalidade que deseja finalizar

git flow feature finish MINHA_FUNCIONALIDADE

4. git flow feature publish

Publica a funcionalidade



5. git flow feature pull

Faz o pull de uma funcionalidade, simplesmente muda de branch

git flow feature pull MINHA_FUNCIONALIDADE

6. git flow release start

Começa uma nova versão, cria uma branch baseada no develop

git flow release start MINHA_VERSAO

7. git flow release publish

Publica a versão

git flow release publish MINHA_VERSAO

8. git flow release finish

Finaliza a versão, executando várias ações: Faz merge do branch MINHA_VERSAO no MASTER; Faz merge do branch MINHA_VERSAO e um checkout no DEVELOP; Etiqueta a versão com seu nome; Remove o branch MINHA_VERSAO;

git flow release finish MINHA_VERSAO

9. git flow hotfix start

Quando surge a necessidade de uma alteração imediata no código, esse comando cria um branch baseado no MAIN marcando a versão defeituosa.

git flow hotfix start MEU_HOTFIX

10. git flow hotfix finish

Finaliza o hotfix, irá fazer um merge no MAIN e no DEVELOP, o merge no MAIN será etiquetado

git flow hotfix finish MEU_HOTFIX
