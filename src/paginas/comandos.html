Comandos


1. Git Clone

Git Clone é um comando usado para baixar a versão mais recente do código fonte de um repositório remoto e salvar em seu computador.

Há algumas maneiras de utilizar, nesse caso vamos exemplificar com HTTPS.

Para clonar, é somente digitar o código abaixo no bash do git e aguardar o download.

git clone <https://link-com-o-nome-do-repositório>


2. Git Branch

Git Branch é muito utilizado no meio de um time de desenvolvimento de software. Utilizando o comando, é criada uma ramificação local, onde é possível publicar uma versão do software para alguma tarefa em específico.

É usado para criar ramificações que geralmente tem seus nomes prés estabelecidos por convenções, como por exemplo, uma branch somente para fazer testes de acréscimos de funcionalidades.

git branch <nome-da-branch>

Com a branch criada localmente, podemos enviar para o repositório remoto com o seguinte comando.

git push <local-remoto> <nome-da-branch>


3. Git Checkout

O Git Checkout é utilizado para 'trocar' de branch. Para conseguir usar uma branch que foi criada, precisamos 'entrar' nela, ai vem a utilização do comando.

Há alguns passos que devem ser seguidos para a troca de branch com sucesso:

  • Não pode haver alterações sem serem salvas na branch, ou seja, deve usar ou o commit ou o stash antes de fazer a troca.
  • A branch que você quer 'entrar' deve existir em seu espaço local.
    • git checkout <nome-da-branch>

    Também temos a possibilidade de criar uma branch e já dar o checkout ao mesmo tempo. Para isso, podemos usar o comando abaixo.

    git checkout -b <nome-da-branch>


    4. Git Status

    O comando Git Status nos fornece todas as informações necessárias sobre a branch atual e os arquivos.

    git status

    Com ele temos:

  • Se a branch atual está atualizada.
  • Se precisamos fazer o commit, push ou pull de alguma informação.
  • Se os arquivos estão em fas de stage, fora dessa fase ou se não estão sendo rastreados.
  • Se há criação, modificação ou exclusão de arquivos.


    • 5. Git Add

    Quando criamos, exlcluimos ou alteramos um arquivo, essas alterações são feitas localmente, não sendo adicionadas sozinha ao próximo commit (a não ser que seja feita alteração das configurações). O Git Add é utilizado para incluir essas alterações de um ou vários arquivos no próximo commit, fazendo com que estes fiquem em 'stage'.

    Para adicionar um único arquivo:

    git add <arquivo>

    Para adicionar todos os arquivos modificados:

    git add -A

    Observação: O comando git add não faz alteração no repositório, só é salvo quando usamos o git commit.



    6. Git Commit

    Após as alterações estarem em stage podemos salva-lás em espaço de trabalho local. O Git Commit é como se criássemos um ponto de restauração para mais tarde. Junto dessa funcionalidade de salvar as alterações que estão em stage, ele adiciona uma mensagem como forma de explicar o que há nessas alterações que foram feitas.

    git commit -m "mensagem do commit"


    7. Git Push

    Após salvar as informações em espaço local, podemos enviar ao servidor remoto. Utilizamos o comando Git Push para realizar o upload dos commits no repositório remoto.

    git push <repositório-remoto> <nome-da-branch>


    8. Git Pull

    O Git Pull é usado para obter as atualizações do repositório remoto. Podemos dizer que esse comando é uma combinação de dois: Git Fetch e Git Merge.

    Quando o git pull é disparado, recebemos as atualizações do repositório remoto(git fetch) e aplicamos imediatamente as alterações em localmente (git merge).

    Essa operação de atualização pode causar alguns conflitos que você deverá resolver manualmente.

    git pull <repositório-remoto>


    9. Git Revert

    Algumas vezes precisamos desfazer alterações, seja por bugs, mudança de escopo, entre outros. Podemos fazer de diversas formas, mas sempre tomando o devido cuidado com os comandos para não fazer exclusões indesejadas.

    Para usar o Git Revert, precisamos descobrir o código hash do commit que queremos desfazer. Usando o comando git log, conseguimos essa informação.

    git log --oneline;

    Após, podemos usar o git revert com o código descoberto.

    git revert <código-hash>

    Depois, você verá uma tela de editor, basta pressionar shift + q para sair:

    Com isso, teremos nossos arquivos com as alterações desfeitas. Esse comando não exclui o commit revertido, ele cria um outro commit.

    Essa funcionalidade permite 'voltarmos no tempo' sem mexer no histórico. Outra medida de segurança está no fato de que tudo acontece em nosso sistema local, a menos que demos um push de tudo para o repositório remoto.



    10. Git Merge

    O Git Merge é utilizado para integrar informações de uma branch para outra. Por exemplo, quando está com tudo certo na branch que estamos desenvolvendo, podemos colocar o código na banch pai (dev ou master/main).

    Primeiro devemos trocar para a branch que vai receber o código:

    git checkout <dev>

    Antes do merge, atualize sua branch dev local:

    git fetch

    Por fim, podemos fazer o merge da sua branch do recurso em dev:

    git merge <nome-da-branch-com-o-recurso>

    Observação: Antes de fazer o merge, certifique que a sua branch dev tenha a versão mais recente. Do contrário, você pode ter que lidar com conflitos.
